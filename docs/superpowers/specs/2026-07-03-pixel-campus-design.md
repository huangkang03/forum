# 像素校园 · 微信剧情小游戏 — 设计规格书

**日期：** 2026-07-03  
**类型：** 像素风校园模拟 + 剧情养成  
**平台：** 微信小游戏 (Cocos Creator 3.x 2D)  
**时长：** 长篇连载，每学段 20-30 分钟，全游戏跨越小学→博士

---

## 一、整体架构

```
Game Entry (SceneBoot)
  ↓
GameManager — 全局状态 & 生命周期
  ├─ currentPhase: 学段枚举 (小学..博士)
  ├─ calendarDay: 当前日历天数
  └─ playerData: 玩家数值 & flag 集合
  ↓
┌─────────────────┬──────────────────────┐
│ 剧情模块 (Story) │ 养成模块 (Calendar)   │
│ StoryManager    │ CalendarManager       │
│ 章节 JSON 驱动   │ 日历推演 & 活动安排    │
│ 分支 & 条件      │ 数值结算 & 随机事件    │
└─────────────────┴──────────────────────┘
  ↓
EventManager — 事件总线，解耦各模块
  ↓
┌──────────┬────────────┬──────────────┐
│ UI 层     │ 数据层      │ 资源层        │
│ Dialogue  │ SaveManager │ 像素素材      │
│ Calendar  │ 本地+云同步  │ textures/    │
│ Status    │ Resource    │ audio/       │
│ Character │ Manager     │ stories/     │
└──────────┴────────────┴──────────────┘
```

**两种游戏模式循环切换：** 剧情模式（手写章节） ↔ 自由模式（日历活动）。学段开始 → 自由期循环（含触发式剧情插入）→ 期末考试 → 学段结束/升学。

## 二、后端 & 数据架构

**后端：微信云开发（WeChat CloudBase）**

| 能力 | 用途 |
|------|------|
| 云数据库 | 玩家数据、好友关系、社交互动 |
| 云存储 | 跨设备存档同步、章节/题库远程加载 |
| 云函数 | 好友申请、考试排名计算、数据校验 |

**数据模型：**

- `players` — 玩家基础信息、游戏进度、五项数值、好友列表
- `saves` — 多档位存档快照，按时间戳解决跨设备冲突
- `friendships` — 好友关系（pending/accepted）
- `social_actions` — 串门留言、送礼物记录
- `exam_records` — 考试得分、排名快照

**加好友：** 分享卡片携带 openid → 对方点击进入 → 发起申请 → 云函数写入 → 双方 friends[] 更新

**跨设备存档：** 本地存档 → 云存储自动同步；新设备启动 → 拉取云端 → 时间戳最新优先合并

## 三、核心玩法循环

```
开始新学段
  ↓
【剧情模式】开学事件（手写章节，强制）
  ↓
【自由模式】← 循环 ──────────────────┐
  ├─ 展示本周日历（周一~日）              │
  ├─ 玩家分配每日活动槽（上/下午/晚上）    │
  ├─ 执行 → 数值结算 + 能量消耗           │
  ├─ 周总结（数值变化 + 解锁标记）         │
  ├─ 触发剧情条件？→【剧情模式】穿插事件   │
  └─ 否则继续 ───────────────────────┘
  ↓
【剧情模式】期末事件 → 考试
  ↓
数值达标检查 → 结局/升学分支
  ↓
进入下一学段 / 游戏结局
```

**一个学段约 8-16 个自由活动周（学段越高周数越多），玩家每周操作约 2-3 分钟。**

## 四、日历 & 活动系统

**每日三个活动槽：** 上午（周末自动休息）/ 下午 / 晚上

| 活动 | 主要数值+ | 副作用 | 能量消耗 |
|------|----------|--------|---------|
| 📚 学习 | 学业 +3~5 | -15 | -15 |
| ⚽ 运动 | 体育 +3~5 | -20 | -20 |
| 🎨 艺术 | 艺术 +3~5 | -10 | -10 |
| 💬 社交 | 社交 +3~5 | 学业随机-1 | -10 |
| 🎮 兴趣 | 兴趣 +3~5 | 随机 | -10 |
| 😴 休息 | 体力 +30 | — | +30 |
| 🔀 特殊 | 视事件 | — | 视情况 |

**核心约束：** 体力低时活动收益减半；周末上午自动休息；特殊日期（考试、活动日）锁定槽位为固定事件。

## 五、社交 & 好友互动

| 互动 | 描述 | 限制 |
|------|------|------|
| 🏫 串门 | 访问好友校园页，留足迹和留言 | 1次/天/人 |
| 🎁 送礼物 | 像素小道具，给好友 buff 加成 | 1次/天/人 |
| 📊 排行榜 | 学业/体育/艺术/社交四项，好友内排名 | 周结算 |

留言支持预设模板 + 自由输入。礼物通过游戏内货币或每日登录获取，收到时玩家获得临时 buff。

## 六、考试 & 真实排行

**考试：** 每学段末强制触发，20 题限时单选，从题库随机抽取。

**题库结构：**
- 按科目 + 难度 + 适用学段分类，JSON 存储
- 内置题库打包发布，远程题库云端更新防作弊
- 题型：单选题为主，少数判断题
- 评分：按难度赋分，云函数计算排名

**排行榜：** 好友榜（微信开放数据域）+ 全服榜（可选前100）。排名直接影响学期结局分支（如「保送」剧情）。

**防作弊：** 每次题目随机打乱顺序、服务器端验证答题时间、以云函数结果为准。

## 七、主角 & 角色系统

**主角创建：** 半自定义 — 玩家选择性别（男/女）+ 输入名字，性格通过游戏内选择逐步塑造。

**青梅竹马角色：**
- 选男主 → 青梅竹马叫「小维」（女性，性格活泼开朗）
- 选女主 → 青梅竹马叫「小康」（男性，性格温和可靠）
- 从小学一年级即出现，贯穿全游戏，是第一个拥有完整个人剧情的核心角色

**角色数据：** 每个角色关联首次出现学段、标签、像素立绘表情集、个人剧情线触发条件。

**好感度等级：**

| 等级 | 数值 | 解锁 |
|------|------|------|
| 陌生人 | 0-19 | 基础对话 |
| 相识 | 20-39 | 日常闲聊 |
| 朋友 | 40-59 | 周末特殊邀约 |
| 好友 | 60-79 | 个人剧情线 |
| 挚友/恋人 | 80+ | 影响结局 |

**角色规模：** 全游戏 15-25 个可互动角色，其中 5-8 个有完整个人剧情线。

## 八、视觉 & 资源规范

- **风格：** 现代像素风，参考星露谷物语
- **分辨率：** 设计分辨率 750×1334
- **角色：** 多表情像素立绘（default/happy/sad/blush 等）
- **场景：** 教室、操场、社团活动室、校园庭院等像素背景
- **UI：** 像素字体，对话框/按钮/日历面板全部像素风格
- **BGM：** 轻松治愈的 chiptune 风格
- **SFX：** 像素风音效（翻书、下课铃、考试钟声）

## 九、技术约束

- **引擎：** Cocos Creator 3.x + TypeScript
- **包体：** 微信小游戏代码包 ≤ 4MB，资源通过远程分包加载
- **性能：** 2D 像素游戏渲染压力低，帧率目标 60fps
- **兼容：** 微信客户端最低版本 7.0+
